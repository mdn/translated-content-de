## Beschreibung

`RegExp.prototype.dotAll` hat den Wert `true`, wenn das `s`-Flag verwendet wurde; andernfalls `false`. Das `s`-Flag gibt an, dass das spezielle Zeichen Punkt (`.`) zusÃ¤tzlich zu den folgenden Zeilentrennzeichen ("newline")-Zeichen in einem String passen soll, die es sonst nicht passen wÃ¼rde:

- U+000A LINE FEED (LF) (`\n`)
- U+000D CARRIAGE RETURN (CR) (`\r`)
- U+2028 LINE SEPARATOR
- U+2029 PARAGRAPH SEPARATOR

Das bedeutet effektiv, dass der Punkt jede UTF-16-Codeeinheit trifft. Es wird jedoch _keine_ Zeichen treffen, die sich auÃŸerhalb der Unicode Basic Multilingual Plane (BMP) befinden, auch bekannt als astrale Zeichen, die durch [Surrogatpaare](/de/docs/Web/JavaScript/Reference/Global_Objects/String#utf-16_characters_unicode_code_points_and_grapheme_clusters) dargestellt werden. Zum Treffen dieser Zeichen sind stattdessen zwei `.`-Muster erforderlich, nicht eines.

```js

// Array(3) [ "ðŸ˜„", "\ud83d", "\ude04" ]
```

Das [`u`](/de/docs/Web/JavaScript/Reference/Global_Objects/RegExp/unicode)-Flag (unicode) kann verwendet werden, um zu erlauben, dass der Punkt astrale Zeichen als ein einzelnes Zeichen trifft.

```js

// Array [ "ðŸ˜„" ]
```

Beachten Sie, dass ein Muster wie `.*` dennoch in der Lage ist, astrale Zeichen als Teil eines grÃ¶ÃŸeren Kontexts _zu konsumieren_, selbst ohne das `u`-Flag.

```js

```
